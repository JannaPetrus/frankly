# Frankly — Учебный план разработки

> Режим обучения: каждая фаза объясняется с нуля — что, зачем и почему именно так.

**Цель:** Построить личный финансовый дашборд, изучив полный цикл современного full-stack приложения на Next.js 14.

---

## Как мы работаем

- Перед каждой фазой — объяснение концепций и причин выбора технологий
- Во время фазы — разбор каждого решения по ходу
- После фазы — проверка и разбор результата
- Код пишем вместе, я объясняю каждый шаг

---

## Фаза 1 — Основа проекта

**Что делаем:** Инициализируем Next.js проект, настраиваем TypeScript, подключаем Prisma, запускаем PostgreSQL через Docker.

**Что изучим:**
- Что такое Next.js App Router и чем он отличается от Pages Router
- Зачем нужен TypeScript и как его настраивать в проекте
- Что такое ORM и зачем нужен Prisma вместо чистого SQL
- Что такое Docker и docker-compose — как они решают проблему "у меня работает, у тебя нет"
- Как описать схему базы данных в Prisma и применить её через миграции

**Результат фазы:** Запущенный Next.js проект с подключённой базой данных PostgreSQL, работающий в Docker.

---

## Фаза 2 — Аутентификация и навигация

**Что делаем:** Подключаем NextAuth.js, создаём страницу входа, делаем layout с навигацией (sidebar на десктопе, tab bar на мобиле).

**Что изучим:**
- Что такое аутентификация и сессии — как браузер "помнит" пользователя
- Что такое NextAuth.js и какие провайдеры он поддерживает (magic link, Google OAuth)
- Что такое middleware в Next.js — как защищать маршруты от неавторизованных пользователей
- Что такое layout в App Router — как один компонент оборачивает несколько страниц
- Как строить адаптивную навигацию с Tailwind (mobile-first подход)

**Результат фазы:** Рабочий вход в приложение, защищённые маршруты, навигация между разделами.

---

## Фаза 3 — Транзакции: CRUD и список

**Что делаем:** Создаём API-маршруты для транзакций, страницу со списком, форму добавления/редактирования, фильтры.

**Что изучим:**
- Что такое REST API и как он устроен (GET, POST, PATCH, DELETE)
- Как работают Route Handlers в Next.js App Router (`/api/...`)
- Что такое Zod и зачем валидировать данные на сервере, а не только на клиенте
- Что такое React Hook Form — как управлять формами без лишнего state
- Как делать фильтрацию данных через query-параметры URL
- Что такое оптимистичные обновления и когда они нужны

**Результат фазы:** Полноценный CRUD для транзакций — можно добавлять, редактировать, удалять, фильтровать записи.

---

## Фаза 4 — Дашборд: метрики и графики

**Что делаем:** Строим главную страницу — карточки с метриками, бар-чарт доходов/расходов за 6 месяцев, пай-чарт по категориям.

**Что изучим:**
- Как агрегировать данные в Prisma (суммы, группировки по месяцам)
- Что такое Server Components в Next.js — когда данные загружаются на сервере, а не в браузере
- Что такое Recharts — как строить интерактивные графики из данных
- Как проектировать API для статистики (отдельные маршруты `/api/stats/...`)
- Как работает кэширование данных в Next.js App Router

**Результат фазы:** Живой дашборд, который показывает реальную картину финансов за текущий месяц.

---

## Фаза 5 — Цели накоплений

**Что делаем:** Страница целей — создание, пополнение, прогресс-бар, прогноз "через сколько месяцев достигнете цели".

**Что изучим:**
- Как считать прогноз на основе текущего темпа накоплений (бизнес-логика)
- Как хранить частичные пополнения цели через связанные данные
- Как строить визуальные компоненты прогресса (progress bar)
- Паттерн "optimistic UI" — обновление интерфейса до ответа сервера

**Результат фазы:** Страница целей с возможностью откладывать деньги и видеть, когда цель будет достигнута.

---

## Фаза 6 — Импорт CSV

**Что делаем:** Кнопка "Загрузить выписку" — загрузка CSV файла, парсинг, предпросмотр, импорт транзакций.

**Что изучим:**
- Как работает загрузка файлов в браузере (File API, FormData)
- Как парсить CSV на сервере (что за формат, edge cases)
- Паттерн "preview before import" — показать пользователю что будет загружено
- Обработка ошибок при импорте (дубли, неверный формат, отсутствующие категории)
- Транзакции в базе данных — как атомарно записать много строк

**Результат фазы:** Возможность загрузить банковскую выписку в CSV и автоматически добавить транзакции.

---

## Фаза 7 — Docker, CI/CD и деплой

**Что делаем:** Пишем Dockerfile для production, настраиваем GitHub Actions pipeline, деплоим на Railway.

**Что изучим:**
- Разница между dev и production окружением — что меняется
- Как написать многоэтапный Dockerfile (multi-stage build) — почему это важно для размера образа
- Что такое GitHub Actions — как автоматизировать проверки и деплой при каждом push
- Что такое Railway — как это работает по сравнению с Heroku/Vercel
- Переменные окружения в production — как безопасно хранить секреты
- Миграции базы данных в production — как не потерять данные при обновлении схемы

**Результат фазы:** Полный автоматический пайплайн: push в main → тесты → сборка → деплой на продакшн.

---

## Фаза 8 — Мобильная полировка

**Что делаем:** Доводим мобильный опыт — тач-управление, оптимизация форм, финальные правки UI.

**Что изучим:**
- Mobile-first подход в Tailwind — breakpoints и когда их применять
- Оптимизация форм для мобильных (`inputmode`, `type="number"`, размеры кнопок)
- Тестирование на реальных устройствах и в DevTools
- Производительность — что замедляет мобильный интерфейс и как это фиксить

**Результат фазы:** Приложение удобно работает на телефоне, не только на десктопе.

---

## Фаза 9 — AI-аналитика с Claude API (после v1)

**Что делаем:** Подключаем Anthropic Claude API для умных подсказок — анализ трат, советы по экономии, прогнозы.

**Что изучим:**
- Что такое Anthropic SDK и как строить запросы к LLM
- Как правильно формировать prompt с финансовыми данными пользователя
- Prompt caching — как снизить стоимость запросов при повторяющихся контекстах
- Streaming-ответы — как показывать ответ AI по мере генерации
- Безопасность: что нельзя отправлять в LLM, как анонимизировать данные

**Результат фазы:** Кнопка "Спросить AI" — пользователь получает персональный анализ своих финансов.

4. **Проверка** — запускаем, смотрим что работает
